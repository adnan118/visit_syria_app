نظرة عامة

تتيح واجهة برمجة التطبيقات (API) الخاصة بالمنشورات للمستخدمين إنشاء المحتوى ومشاركته والتفاعل معه. تشمل الميزات:

  • إنشاء وتعديل المنشورات مع دعم الوسائط (صور ومقاطع فيديو)
  • عرض المنشورات مع تصفية وترقيم الصفحات
  • التفاعل مع المنشورات (إعجاب، تعليق، مشاركة، حفظ)
  • إدارة رؤية المنشورات (إخفاء/إظهار)
  • تتبع الإحصائيات (المشاهدات، الإعجابات، التعليقات، المشاركات)
  • الوصول إلى المنشورات حسب الاهتمامات

هيكل المنشور

كل منشور يحتوي على الحقول التالية:

الحقل النوع الوصف
id / _id رقم المعرف الفريد للمنشور
author كائن معلومات المؤلف (id, firstName, lastName, username, image, isActive, imageUrl)
title نص عنوان المنشور
content نص محتوى المنشور
media مصفوفة الوسائط المرفقة (صور ومقاطع فيديو)
tags مصفوفة معرفات الوسوم المرتبطة بالمنشور
services مصفوفة معرفات الخدمات المرتبطة بالمنشور
likesCount رقم عدد الإعجابات
commentsCount رقم عدد التعليقات
sharesCount رقم عدد المشاركات
viewsCount رقم عدد المشاهدات
isDeleted boolean حالة الحذف (мяك )
status نص حالة المنشور (pending, approved, rejected)
rejectionReason نص سبب رفض المنشور (عند الحالة rejected)
createdAt تاريخ تاريخ الإنشاء
updatedAt تاريخ تاريخ آخر تحديث

إنشاء منشور

نقطة النهاية: POST /api/v1/posts

إنشاء منشور جديد مع دعم رفع الصور ومقاطع الفيديو.

معلمات الطلب

الحقل النوع مطلوب الوصف
title نص نعم عنوان المنشور
content نص نعم محتوى المنشور
tags مصفوفة أو نص لا معرفات الوسوم (مفصولة بفواصل أو مصفوفة)
services مصفوفة أو نص لا معرفات الخدمات (مفصولة بفواصل أو مصفوفة)
images ملفات لا الصور المرفقة (تستبدل الوسائط القديمة تلقائياً)
video ملف لا مقطع الفيديو المرفق (يستبدل الوسائط القديمة تلقائياً)

مثال على الطلب

curl -X POST "https://visitsyria.fun/api/v1/posts" \
  -H "Authorization: Bearer <access_token>" \
  -F "title=عنوان المنشور" \
  -F "content=محتوى المنشور" \
  -F "tags=1,2,3" \
  -F "media=@image1.jpg" \
  -F "media=@video1.mp4"

عرض المنشورات

نقطة النهاية: GET /api/v1/posts

عرض المنشورات العامة مع تصفية وترقيم الصفحات.

معلمات الاستعلام

الحقل النوع الافتراضي الوصف
page رقم 1 رقم الصفحة
limit رقم 20 عدد المنشورات في الصفحة
tags نص - تصفية حسب معرفات الوسوم (مفصولة بفواصل)
services نص - تصفية حسب معرفات الخدمات (مفصولة بفواصل)
q نص - بحث في العنوان والمحتوى

مثال على الطلب

curl -X GET "https://visitsyria.fun/api/v1/posts?page=1&limit=10&tags=1,2&q=البحث" \
  -H "Authorization: Bearer <access_token>"

مثال على الاستجابة

{
  "status": "success",
  "data": [
    // مصفوفة من المنشورات
  ],
  "pagination": {
    "currentPage": 1,
    "totalPages": 5,
    "totalPosts": 45,
    "hasNext": true,
    "hasPrev": false,
    "pageSize": 10
  }
}

تحديث منشور

نقطة النهاية: PUT /api/v1/posts/:id

تحديث منشور موجود (للمؤلف أو المدير فقط). عند تحميل وسائط جديدة، يتم استبدال الوسائط القديمة تلقائياً.

معلمات الطلب

الحقل النوع مطلوب الوصف
title نص لا عنوان المنشور
content نص لا محتوى المنشور
tags مصفوفة أو نص لا معرفات الوسوم (مفصولة بفواصل أو مصفوفة)
services مصفوفة أو نص لا معرفات الخدمات (مفصولة بفواصل أو مصفوفة)
images ملفات لا الصور المرفقة (تستبدل الوسائط القديمة تلقائياً)
video ملف لا مقطع الفيديو المرفق (يستبدل الوسائط القديمة تلقائياً)

مثال على الطلب

curl -X PUT "https://visitsyria.fun/api/v1/posts/123" \
  -H "Authorization: Bearer <access_token>" \
  -F "title=العنوان المحدث" \
  -F "content=المحتوى المحدث" \
  -F "tags=1,4,5" \
  -F "images=@new_image.jpg" \
  -F "video=@new_video.mp4"

حذف منشور

نقطة النهاية: DELETE /api/v1/posts/:id

حذف منشور (للمؤلف أو المدير فقط).

مثال على الطلب

curl -X DELETE "https://visitsyria.fun/api/v1/posts/123" \
  -H "Authorization: Bearer <access_token>"

مثال على الاستجابة

{
  "status": "success",
  "message": "Post deleted successfully"
}

تفاعلات المنشور

إضافة تعليق

نقطة النهاية: POST /api/v1/posts/:id/comments

الحقل النوع مطلوب الوصف
content نص نعم محتوى التعليق
parent رقم لا معرف التعليق الأصلي (للردود)

مشاركة المنشور

نقطة النهاية: POST /api/v1/posts/:id/share

زيادة عدد المشاركات وعرض روابط المشاركة.

إعجاب/إزالة إعجاب

نقطة النهاية: POST /api/v1/likes/:postId/toggle

تبديل حالة الإعجاب بالمنشور.

حفظ/إزالة حفظ

نقطة النهاية: POST /api/v1/saves/:postId/toggle

تبديل حالة حفظ المنشور.

رؤية المنشور

إخفاء/إظهار المنشور

نقطة النهاية: POST /api/v1/posts/:id/toggle-hidden

تبديل حالة رؤية المنشور (мяك ) للمؤلف فقط.

عرض منشوراتي

نقطة النهاية: GET /api/v1/posts/me

عرض جميع منشوراتي (تشمل المخفية).

إحصائيات المنشور

يتم تتبع الإحصائيات التالية تلقائياً:

  • مشاهدات (viewsCount): تزداد تلقائياً عند عرض المنشور
  • إعجابات (likesCount): تزداد/تنقص عند تفاعل المستخدمين
  • تعليقات (commentsCount): تزداد/تنقص عند إضافة/حذف تعليق
  • مشاركات (sharesCount): تزداد عند مشاركة المنشور

المنشورات الأعلى

نقطة النهاية: GET /api/v1/posts/best/top

عرض المنشورات الأعلى حسب معايير مختلفة.

معلمات الاستعلام

الحقل القيم الافتراضي الوصف
metric views, likes, comments, share views المعيار للترتيب
period day, week, month, year, all day الفترة الزمنية

مثال على الطلب

curl -X GET "https://visitsyria.fun/api/v1/posts/best/top?metric=likes&period=week" \
  -H "Authorization: Bearer <access_token>"

المنشورات حسب الاهتمامات

نقطة النهاية: GET /api/v1/posts/me/interests

عرض المنشورات المرتبطة باهتمامات المستخدم.

مثال على الطلب

curl -X GET "https://visitsyria.fun/api/v1/posts/me/interests" \
  -H "Authorization: Bearer <access_token>"

معاينة المنشور (Open Graph)

نقطة النهاية: GET /api/v1/posts/:id/og

عرض معاينة المنشور بصيغة HTML مع بيانات Open Graph للمشاركة على وسائل التواصل الاجتماعي.
