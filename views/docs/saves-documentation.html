نظرة عامة

تتيح واجهة برمجة التطبيقات (API) الخاصة بالحفظ للمستخدمين حفظ المنشورات المفضلة لديهم وإدارتها. تشمل الميزات:

  • حفظ المنشورات وإلغاء الحفظ
  • عرض قائمة بالمنشورات المحفوظة
  • تبديل حالة الحفظ تلقائياً

هيكل الحفظ

كل عملية حفظ تحتوي على الحقول التالية:

الحقل النوع الوصف
id رقم المعرف الفريد لعملية الحفظ
postId رقم معرف المنشور المحفوظ
userId رقم معرف المستخدم الذي قام بالحفظ
createdAt تاريخ تاريخ الحفظ
updatedAt تاريخ تاريخ آخر تحديث لعملية الحفظ

حفظ/إلغاء حفظ منشور

نقطة النهاية: POST /api/v1/saves/:postId/toggle

حفظ منشور أو إلغاء حفظه (للمستخدمين المسجلين فقط).

مثال على الطلب

curl -X POST "https://visitsyria.fun/api/v1/saves/123/toggle" \
  -H "Authorization: Bearer <access_token>"

مثال على الاستجابة (عند الحفظ)

{
  "status": "success",
  "message": "Post saved successfully",
  "data": {
    "saved": true
  }
}

مثال على الاستجابة (عند إلغاء الحفظ)

{
  "status": "success",
  "message": "Post unsaved successfully",
  "data": {
    "saved": false
  }
}

عرض المنشورات المحفوظة

نقطة النهاية: GET /api/v1/saves/me

عرض قائمة بالمنشورات المحفوظة من قبل المستخدم الحالي.

معلمات الاستعلام

الحقل النوع الافتراضي الوصف
page رقم 1 رقم الصفحة
limit رقم 20 عدد المنشورات في الصفحة

مثال على الطلب

curl -X GET "https://visitsyria.fun/api/v1/saves/me?page=1&limit=10" \
  -H "Authorization: Bearer <access_token>"

مثال على الاستجابة

{
  "status": "success",
  "data": [
    // مصفوفة من المنشورات المحفوظة
  ],
  "pagination": {
    "currentPage": 1,
    "totalPages": 5,
    "totalSaved": 45,
    "hasNext": true,
    "hasPrev": false,
    "pageSize": 10
  }
}
