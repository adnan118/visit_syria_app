نظرة عامة

يوفر تطبيق زيارة سوريا نظام إدارة شامل للمستخدمين يشمل:

  1. إدارة الملف الشخصي - تحديث البيانات الشخصية وصورة الملف الشخصي
  2. إدارة الاهتمامات - إضافة وإزالة الاهتمامات (الوسوم)
  3. إدارة المستخدمين من قبل الإدارة - عرض وتحديث وحذف المستخدمين

نقاط نهاية المستخدمين

GET /api/v1/users/:id - عرض ملف شخصي عام
PUT /api/v1/users/:id - تحديث الملف الشخصي
DELETE /api/v1/users/:id - حذف المستخدم
GET /api/v1/users/:userId/interests - عرض اهتمامات المستخدم
POST /api/v1/users/:userId/interests - إضافة اهتمام للمستخدم
DELETE /api/v1/users/:userId/interests/:tagId - إزالة اهتمام من المستخدم
PUT /api/v1/users/:userId/interests - تحديث جميع اهتمامات المستخدم

نقاط نهاية الإدارة

GET /api/v1/admin/users - عرض جميع المستخدمين (للإدارة)
GET /api/v1/admin/users/:id - عرض مستخدم محدد (للإدارة)
PUT /api/v1/admin/users/:id - تحديث بيانات المستخدم (للإدارة)
DELETE /api/v1/admin/users/:id - حذف مستخدم (للإدارة)

إدارة الملف الشخصي

عرض الملف الشخصي

نقطة النهاية: GET /api/v1/users/:id

عرض ملف شخصي عام باستخدام المعرف الرقمي أو اسم المستخدم.

مثال على الطلب

curl -X GET "https://visitsyria.fun/api/v1/users/1" \
  -H "Authorization: Bearer <access_token>"

مثال على الاستجابة

{
  "status": "success",
  "data": {
    "id": 1,
    "firstName": "محمد",
    "lastName": "أحمد",
    "username": "محمد أحمد",
    "email": "mohammad@example.com",
    "mobile": "0912345678",
    "bio": "مطور برمجيات من سوريا",
    "image": "user_1234567890.jpg",
    "imageUrl": "https://visitsyria.fun/uploads/users/user_1234567890.jpg",
    "createdAt": "2025-01-15T10:30:00.000Z",
    "updatedAt": "2025-01-20T14:45:00.000Z"
  }
}

تحديث الملف الشخصي

نقطة النهاية: PUT /api/v1/users/:id

تحديث بيانات الملف الشخصي ورفع صورة جديدة.

معلمات الطلب

الحقل النوع مطلوب الوصف
firstName نص لا الاسم الأول للمستخدم (2-50 حرف)
lastName نص لا اسم العائلة للمستخدم (2-50 حرف)
username نص لا اسم المستخدم (3-30 حرف، أحرف وأرقام ونقاط وشرطات)
email نص لا عنوان بريد إلكتروني صالح
mobile نص لا رقم الجوال (9-15 رقماً)
bio نص لا نبذة عن المستخدم
image ملف لا صورة الملف الشخصي (JPG, PNG, WEBP)

مثال على الطلب

curl -X PUT "https://visitsyria.fun/api/v1/users/1" \
  -H "Authorization: Bearer <access_token>" \
  -F "firstName=محمد" \
  -F "lastName=أحمد" \
  -F "bio=مطور برمجيات من سوريا" \
  -F "image=@profile.jpg"

مثال على الاستجابة

{
  "status": "success",
  "message": "User updated successfully",
  "data": {
    "id": 1,
    "firstName": "محمد",
    "lastName": "أحمد",
    "username": "محمد أحمد",
    "email": "mohammad@example.com",
    "mobile": "0912345678",
    "bio": "مطور برمجيات من سوريا",
    "image": "user_1234567890.jpg",
    "imageUrl": "https://visitsyria.fun/uploads/users/user_1234567890.jpg",
    "createdAt": "2025-01-15T10:30:00.000Z",
    "updatedAt": "2025-01-20T15:00:00.000Z"
  }
}

اهتمامات المستخدمين

عرض اهتمامات المستخدم

نقطة النهاية: GET /api/v1/users/:userId/interests

عرض جميع الاهتمامات (الوسوم) المرتبطة بالمستخدم.

مثال على الطلب

curl -X GET "https://visitsyria.fun/api/v1/users/1/interests" \
  -H "Authorization: Bearer <access_token>"

مثال على الاستجابة

{
  "status": "success",
  "data": [
    {
      "id": 1,
      "name": "تاريخ سوريا",
      "description": "كل ما يتعلق بالتاريخ السوري القديم والحديث"
    },
    {
      "id": 5,
      "name": "美食文化",
      "description": " Syrian cuisine and culinary traditions"
    }
  ]
}

إضافة اهتمام للمستخدم

نقطة النهاية: POST /api/v1/users/:userId/interests

ربط اهتمام (وسم) بالمستخدم.

معلمات الطلب

الحقل النوع مطلوب الوصف
tagId رقم نعم معرف الوسم المراد إضافته

مثال على الطلب

curl -X POST "https://visitsyria.fun/api/v1/users/1/interests" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <access_token>" \
  -d '{"tagId": 1}'

مثال على الاستجابة

{
  "status": "success",
  "message": "Interest added successfully"
}

إزالة اهتمام من المستخدم

نقطة النهاية: DELETE /api/v1/users/:userId/interests/:tagId

إزالة اهتمام (وسم) من المستخدم.

مثال على الطلب

curl -X DELETE "https://visitsyria.fun/api/v1/users/1/interests/1" \
  -H "Authorization: Bearer <access_token>"

مثال على الاستجابة

{
  "status": "success",
  "message": "Interest removed successfully"
}

تحديث جميع اهتمامات المستخدم

نقطة النهاية: PUT /api/v1/users/:userId/interests

تحديث جميع اهتمامات المستخدم (استبدال القائمة الحالية).

معلمات الطلب

الحقل النوع مطلوب الوصف
tagIds مصفوفة أرقام نعم معرفات الوسوم الجديدة للمستخدم

مثال على الطلب

curl -X PUT "https://visitsyria.fun/api/v1/users/1/interests" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <access_token>" \
  -d '{"tagIds": [2, 7, 12]}'

مثال على الاستجابة

{
  "status": "success",
  "message": "User interests updated successfully"
}

إدارة المستخدمين من قبل الإدارة

عرض جميع المستخدمين

نقطة النهاية: GET /api/v1/admin/users

عرض قائمة بجميع المستخدمين مع دعم التصفح والترتيب والتصفية.

معلمات الاستعلام

الحقل النوع الافتراضي الوصف
page رقم 1 رقم الصفحة
limit رقم 20 عدد المستخدمين في الصفحة (1-100)
sortBy نص createdAt حقل الترتيب (firstName, lastName, username, email, createdAt, isActive, isAdmin)
sortOrder نص desc ترتيب الصعود أو النزول (asc, desc)
search نص - بحث في الاسم الأول، الاسم الأخير، اسم المستخدم، البريد الإلكتروني، أو الجوال
isActive boolean - تصفية حسب حالة الحساب
isAdmin boolean - تصفية حسب صلاحيات الإدارة

مثال على الطلب

curl -X GET "https://visitsyria.fun/api/v1/admin/users?page=1&limit=10&sortBy=firstName&sortOrder=asc" \
  -H "Authorization: Bearer <access_token>"

اعتبارات الأمان

صلاحيات الوصول

  • المستخدمون العاديون يمكنهم فقط:
    • عرض ملفهم الشخصي وملفات الآخرين
    • تحديث ملفهم الشخصي
    • إدارة اهتماماتهم
    • حذف حسابهم
  • المشرفون يمكنهم:
    • تنفيذ جميع عمليات المستخدم العادي
    • عرض جميع المستخدمين
    • تحديث بيانات أي مستخدم
    • حذف أي مستخدم
    • تغيير صلاحيات الإدارة والحالة

التحقق من الهوية

  • جميع نقاط النهاية (عدا عرض الملف الشخصي العام) تتطلب رمز وصول صالح
  • نقاط نهاية الإدارة تتطلب رمز وصول مع صلاحية isAdmin: true
  • يتم التحقق من أن المستخدم يمكنه فقط تعديل بيانات حسابه (عدا المشرفين)

أمان الملفات

  • يتم ضغط صور الملف الشخصي تلقائياً لتحسين الأداء
  • يتم التحقق من نوع الملف وحجمه قبل الرفع
  • يتم حذف الصور القديمة تلقائياً عند رفع صور جديدة
  • يتم تخزين الملفات في مجلد منظم حسب النوع
